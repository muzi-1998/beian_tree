# 项目文件说明 —— §1.1 时间底座与非平稳特征解析

> 本文档逐文件梳理 **`Project 1 Data Quality Assessment / 1.1 Decomposition`** 子项目中每个文件/目录的作用。
> 项目目标：把北岸污水厂多源异构数据（分钟级 DO/ORP/QR/QIR + 小时级进出水水质）统一转化为
> **去趋势、去周期、并经 ARMA/GARCH 白化的纯净残差 / 创新序列**，作为后续 1.2 节数据质量评估的合法输入。
> 全部实现严格对应 `plan and report/` 下的实施方案 v3.0。

### 3.4 `src/baseline/` —— 趋势/周期分解

### 3.5 `src/whiten/` —— ARMA/GARCH 白化（快慢轨）

| 文件 | 内容 |
|---|---|
| [src/whiten/model_selection.py](src/whiten/model_selection.py) | **逐通道数据驱动模型选择层**(plan §4.3 升级)。在 {ARMA, ARIMA(d), SARIMA(D,s), ARFIMA(分数 d), 局部水平≡ARIMA(0,1,1)} 中自动选择:整数差分阶 d 由 **ADF+KPSS 联合判定**、季节 D 由季节单位根、分数阶由 **LRD 检验(GPH d̂ / R-S Hurst / log-log ACF)** 闸门;统一折叠为组合 AR 多项式 `A(B)=φ(B)·∇(B)`(ARFIMA 用截断分数差分 FIR,在线可递归),现有白化器无需改结构。护栏:过差分/过建模(创新 `acf1<−0.30`)与方差膨胀即拒;差分模型须把窗口 LB 通过率较 ARMA 提升余量才选用(防漂移故障被白化掉)。`gph_d`/`rs_hurst`/`loglog_acf_slope`/`choose_d`/`choose_D`/`fracdiff_weights`/`fit_candidate`/`select_model`。 |
| [src/whiten/offline_identify.py](src/whiten/offline_identify.py) | **慢轨「统计大脑」**(ARMA-only 路径,被 validate.py 消融与 `model_selection.enable:false` 时使用)。在去周期残差上：AIC 网格搜索 ARMA(p,q)（BIC 破平）→ 拟合 ARIMA → ARCH-LM 显著时叠加 GARCH(1,1) → 构建预热内部状态 → 打包不可变 `WhitenModel`。定阶在限点窗口上跑以保证可解性。 |
| [src/whiten/online_whitener.py](src/whiten/online_whitener.py) | **快轨冻结系数滤波**。`OnlineWhitener.step` 每步 O(p+q) 计算标准化创新 η_t 与 GARCH σ²_t（毫秒级，逐分钟运行）；系数**冻结、不在线自适应**（自适应会抹平 drift 故障）；仅当慢轨发布新版本时原子热替换+载入热态。`whiten_series` 用 `scipy.signal.lfilter` 向量化批处理整段序列（离线等价）。 |
| [src/whiten/diagnostics.py](src/whiten/diagnostics.py) | **平稳性与白噪声诊断**（另含 `censored_robust_z` 截尾感知标准化 / `sign_bias_test` Engle-Ng 不对称检验 / `robust_z` 兜底）。Ljung-Box（残差自相关）、ADF（拒单位根）、KPSS（不拒平稳）、ARCH-LM（条件异方差→启用 GARCH）；大 n 抽样封顶以提速。关键 `windowed_lb_pass_rate`：把 36 万点序列切窗逐段 LB 求**通过率**（单次 LB 在大 n 必拒，故用通过率作白化充分性度量）；另有 `acf`/`acf1`/`mean_abs_acf`/`acf_conf` 供出图与衰减度量。 |
| [src/whiten/acceptance_gate.py](src/whiten/acceptance_gate.py) | **换入前接受门**。发布新模型前须通过：①AR/MA 根严格在单位圆内（平稳可逆）②自身创新 LB 通过 ③创新方差相对残差方差不离谱。否则保留旧有效模型。返回 `(passed, reasons)`。 |
| [src/whiten/warmup.py](src/whiten/warmup.py) | **预热热态交接**。换系数若不重置 eps/eta/σ² 缓冲会产生切换尖峰被误判。发布前用新系数在最近 `warmup_hours` 真实残差上重跑，得到一致热态写入 `WhitenModel.warmup_state`。 |
| [src/whiten/param_store.py](src/whiten/param_store.py) | **版本化参数 + 原子热替换**。`WhitenModel` 冻结数据类（version/p/q/ar/ma/intercept/garch/warmup_state/diagnostics）；`ParamStore` 进程内线程安全字典，快轨读不可变快照、慢轨发布新模型即引用交换（GIL 原子），单厂 14 通道规模无需 Redis。 |

> **为何分钟级图分"窗口 + 全程概览"两套**：分解/白化/全部表格诊断都用**完整 36.8 万点序列**;只有"画图"环节,分钟级因点密度过高(比逐日数据密 ~250 倍)无法在一格里既画全程又看清结构,故拆为「10 天窗口(看 24h 周期/白化细节)」+「全程逐日包络(看 256 天全貌与季节演变)」两套。小时级(进出水)点少,直接全程单图。

### 5.5 `outputs/plot_data/` —— 绘图数据包（git 忽略，可再生成）
每张堆叠/组合图对应一对 `<name>.csv`（实际绘图序列，列 `x`=ISO 时间戳）+ `<name>.json`（渲染规格）。配合根目录 [plot_data/replot.py](plot_data/replot.py) 可在不重跑分解白化的前提下，从数据包逐位复现/改版任意图件（详见 [plot_data/README.md](plot_data/README.md)）。
