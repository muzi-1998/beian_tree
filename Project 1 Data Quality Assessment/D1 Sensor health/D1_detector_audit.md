# D1 传感器健康 — 完整逐检测器审计清单

> 审计对象:`D1 Sensor health` v1.1(`run_v11_pipeline.py` + `src/detectors/*`)。
> 审计角度:**残差是否白化、时间序列分解是否准确、各检测器的统计假设是否成立**。
> 评级:❌ 严重(结论可能系统性失真) · ⚠️ 中等(有偏但可用/可缓解) · ✅ 稳健(设计上不受影响)。
> 关键背景:**D1 全程无残差白化**(全仓 `grep arma|garch|whiten|ljung|innovation` 零命中),`resid_min = deperiodise(df_min)` → `resid_h = resid_min.resample("1h").mean()` → 直接喂检测器。§1.1 Decomposition 已产出更准的分解 + 白化创新 + `whiteness_manifest` 契约,本审计的修复主线就是**让 D1 消费 §1.1 的交付物并按 `scoring_mode` 分支**。

---

## 0. 前置层审计(分解 + 白化)

### 0.1 去周期分解 `src/baseline/deperiodise.py`
| 项 | 现状 | 评级 |
|---|---|---|
| 谐波分解 | 日(1440)+周(10080),**固定 3 阶**;系数**只在前 30 天因果拟合、全程套用** | ⚠️ |
| 缺陷①阶数固定 | 非自适应(§1.1 已升级为嵌套-F 自适应选阶) | ⚠️ |
| 缺陷②系数不滚动 | 追不上季节漂移的日幅相,对**方波型曝气 DO** 尤其失真(§1.1 加了分块滚动重拟合 + STL 精修非正弦日形) | ⚠️ |
| 缺陷③`stl_decomposition` | 相位分箱 `groupby(bin_id).transform("mean")` **对整条序列**求均值 → **未来信息泄漏(非因果)** | ❌(若用于流式) |
| 缺陷④ | 无分钟/小时差异化、无出水左截尾、无分解充分性闸门 | ⚠️ |

### 0.2 残差白化 —— **完全缺失** ❌(本审计的根因)
- D1 没有任何 ARMA/GARCH/白化步骤;喂给检测器的是**强自相关残差**(§1.1 实测 hourly 残差 acf1 0.4–0.97)。
- 任何**假设窗内 i.i.d./特定零分布**的检测器,在自相关残差上**有效样本量 n_eff ≪ n**,固定临界值/惩罚项偏小 → **系统性假阳性 / 过度切分**。

## 1. 逐检测器审计

### ① Spike — Hampel `detectors/spike_hampel.py`
- **方法**:滚动中位数 + MAD,`robust_z=|x−med|/(1.4826·MAD) > k(=3)`。
- **输入**:**原始分钟** `df_min`(pipeline `run_spike_detector(df_min,…)`)。
- **假设**:仅"局部对称、尖峰为局部离群"。**不假设全局 i.i.d.**。
- **评级**:✅ **稳健**(局部 MAD 自适应跟踪局部水平与季节,自相关不致命)。
- **缺陷**:⚠️ `rolling(center=True)` → **用到未来点(非因果)**,与"streaming 层"声明不符(批处理回溯无碍,在线即泄漏)。
- **修复**:`center=True → 因果尾随窗`(或保留 center 仅用于离线复盘并注明)。无需白化。

### ② Step — 相邻 KS `detectors/step_adjacent_ks.py :: AdjacentKSStepDetector`
- **方法**:相邻 24h 窗两样本 KS,`D > ks_crit = c_α·√(2/win_h)`。
- **输入**:**未白化** hourly 残差 `resid_h`。
- **假设**:**KS 临界值假设窗内 i.i.d.**(`√(2/n)` 的零分布)。
- **评级**:❌ **严重**。自相关下 n_eff≪win_h → 真实 KS 零分布更宽 → `ks_crit` 偏小 → **阶跃假阳性膨胀**(典型大-n/自相关陷阱)。
- **修复**:(a) 在**白化创新**上做 KS(`whiteness_manifest.scoring_mode==iid` 的通道)→ 临界值此时合法;(b) 否则临界值按 **n_eff 校正**(`ks_crit·√(win/n_eff)`,n_eff 由 manifest `n_eff_ratio` 给)或**块自助**标定;(c) `autocorr_aware` 通道改倚重多变量(④/⑤)。

### ③ Step 辅助 — Page-Hinkley `step_adjacent_ks.py :: PageHinkleyDetector`
- **方法**:CUSUM,参照=**扩张均值**,`ph=cumsum−min > λ(=5)`,`δ=0.5`。
- **输入**:`resid_h`(未白化)。
- **假设**:残差近 i.i.d. 围绕参照;`δ/λ` 为**绝对值**(未按通道 σ 归一)。
- **评级**:⚠️ **中等**。自相关使 CUSUM 累积更快 → 假报增多;绝对 `δ/λ` 跨通道不可比。
- **修复**:`δ/λ` 改为 **σ 倍数**(用残差/创新 robust σ);在白化创新上运行;或参照改稳健尾随中位数。

### ④ Drift — PLS 虚拟传感器 `detectors/drift_pls.py`
- **方法**:用**工程同伴**(同池相邻 + 孪生池 + QR/QIR 驱动)PLS 预测目标,`resid=y−ŷ`,`|resid|/σ_y > 3`。
- **输入**:`resid_h`(多变量)。
- **假设**:**多变量空间冗余**——不依赖单变量白噪声(正是 §1.1 给近单位根通道推荐的 D7 思路)。
- **评级**:✅ **稳健(本组最佳设计)**。
- **缺陷**:⚠️ ① 归一用 `σ_y=std(y)` 而非 `std(resid)`(line 113)——量纲不当(偏保守);② `ffill().fillna(0)` 用 0 填缺失同伴/目标 → **伪值**污染;③ PLS 残差本身可能自相关,固定 3σ 仍轻度过报;④ 仅前 21 天训练、不重拟合。
- **修复**:归一改 `std(resid)`;缺失改插值/掩蔽而非填 0;3σ 可按 PLS 残差 n_eff 微调;可周期重拟合同伴关系(注意别把真漂移学进去)。

### ⑤ Drift 辅助 — FF-PCA `detectors/ffpca_aux.py`
- **方法**:遗忘因子 PCA,SPE(平方重构误差)+ 逐通道贡献,`SPE > p99(训练)`。
- **输入**:`resid_h` 全矩阵(多变量)。
- **假设**:多变量子空间;**不依赖单变量白化**。
- **评级**:✅ **稳健**(辅助"第二意见")。
- **缺陷**:⚠️ ① `fillna(0)` 伪值入协方差;② SPE 本身自相关 → p99 UCL 的逐时假报率非名义 1%(超限聚集),但作 aux 可接受。
- **修复**:NaN 掩蔽而非填 0;UCL 可用块/EWMA 校正。

### ⑥ Freeze — 复合规则 `detectors/freeze_rules.py`
- **方法**:RLE 等值连段 + 低方差(vs 30 天参照 σ)+ 唯一值比;`run≥15 | rel_var<0.05 | uq<0.10`。
- **输入**:**原始分钟** `df_min`。
- **假设**:仅"卡死=不变/低方差/重复值"。**与白化无关**。
- **评级**:✅ **稳健**。
- **缺陷**:⚠️ 阈值(15/0.05/0.10)为启发式;`eps=1e-6` 等值判定对量化/有效位敏感。
- **修复**:阈值按通道噪声基线标定即可;无需白化。

### ⑦ Freeze 辅助 — 响应缺失 `detectors/freeze_response_loss.py`
- **方法**:驱动(QR/QIR)突变后,目标(DO/ORP)应在响应窗内动;否则(方差<p5 且 增益<p10)记响应缺失。
- **输入**:**原始分钟**(目标 + 驱动,多变量因果)。
- **假设**:**驱动→目标因果响应**;不依赖白化。
- **评级**:✅ **稳健**(优秀的多变量机理检验)。
- **缺陷**:⚠️ p5/p10 启发式;驱动事件阈 10% 相对变化对低流量段敏感。
- **修复**:基准窗用"可信小时"集合(已支持 `benchmark_window`);对 QR 负流量段先按 §1.1 打标剔除再算。

### ⑧ Regime — 两层 W1+KS `detectors/regime_two_tier.py`
- **方法**:Tier-1 = 7 天窗 vs 30 天参照的 **Wasserstein W1**(自助基线 99.5%);Tier-2 = 7 天相邻 **KS**;**两层 AND** 才报。
- **输入**:`resid_h`(未白化)。
- **假设**:Tier-1 自助**无放回随机子采样**(破坏时间结构);Tier-2 KS 固定 i.i.d. 临界值。
- **评级**:⚠️ **中等**(AND 门显著缓解)。但:① 自助基线是**非块**子采样 → 低估"两个**连续**7 天窗"应有的 W1(自相关/慢漂移使连续窗差异更大)→ Tier-1 偏过检;② Tier-2 KS 同 ② 的 i.i.d. 问题。
- **修复**:Tier-1 改**块自助**(连续块重采样)标定基线;Tier-2 同 ② 用白化创新或 n_eff 校正;AND 门保留。

### ⑨ PELT 批量变点 `detectors/pelt_batch.py`
- **方法**:L2 代价 + `penalty = 1.5·log(n)·var(x)`(BIC 式),分段。
- **输入**:`resid_h`(未白化),离线每 6h 标定"step 窗"。
- **假设**:**L2 代价 + BIC 惩罚假设段内 i.i.d. 高斯**。
- **评级**:❌ **严重**。自相关下惩罚相对偏小 → **过度切分**(伪变点)→ 写入 blackboard 的 `pelt_step_window` 过多,反过来误掩 drift 冷却。
- **修复**:(a) 在**白化创新**上跑 PELT;(b) 否则惩罚按 **n_eff 放大**(`penalty·(1+ρ)/(1−ρ)`)或改用**自相关感知代价**(如 AR 残差代价);(c) `autocorr_aware` 通道少依赖 PELT,改用 ④/⑤/⑧。

## 2. 缺陷汇总(按严重度)

| 检测器 | 输入 | 受"无白化"影响 | 评级 | 主因 |
|---|---|---|---|---|
| ② Step-KS | 未白化 resid_h | **大** | ❌ | KS i.i.d. 临界值 |
| ⑨ PELT | 未白化 resid_h | **大** | ❌ | BIC 惩罚 i.i.d. → 过切分 |
| ③ Page-Hinkley | 未白化 resid_h | 中 | ⚠️ | CUSUM 累积 + 绝对阈 |
| ⑧ Regime(W1+KS) | 未白化 resid_h | 中(AND 缓解) | ⚠️ | 非块自助 + KS i.i.d. |
| ④ Drift-PLS | resid_h(多变量) | 小 | ✅ | 多变量稳健;σ_y/fillna 小瑕 |
| ⑤ FF-PCA | resid_h(多变量) | 小 | ✅ | 多变量;fillna/SPE-UCL 小瑕 |
| ① Spike-Hampel | 原始 min | 无 | ✅ | 局部稳健;`center=True` 泄漏 |
| ⑥ Freeze 复合 | 原始 min | 无 | ✅ | 启发式阈值 |
| ⑦ Freeze 响应缺失 | 原始 min(多变量) | 无 | ✅ | 启发式 p5/p10 |

**共性问题(跨检测器)**:
1. **无白化**(根因,影响 ②③⑧⑨)。
2. **非因果泄漏**:① Hampel `center=True`;0.1 节 STL 相位分箱用整条序列。
3. **`fillna(0)` 伪值**:④ ⑤(缺失填 0 污染统计)。
4. **绝对阈未按 σ 归一**:③ Page-Hinkley。
5. **分解欠精**:固定 3 阶、系数不滚动(0.1 节)。

## 3. 修复优先级 + §1.1 对接方案

**架构主线**:D1 输入从"自带 deperiodise + 未白化残差"换成 **§1.1 的 `residual_*/innovation_*.parquet` + `whiteness_manifest.csv`**,并按 `scoring_mode` 分支:
- `iid`(已白化创新):**②Step-KS / ⑨PELT / ⑧Tier-2 KS 跑在创新上 → 临界值/惩罚此时合法**。
- `autocorr_aware`(robust_z,近单位根):**不**跑 i.i.d. 的 KS/PELT;漂移走 **④Drift-PLS + ⑤FF-PCA + D7 沿程梯度**,变点/阶跃用 n_eff 校正或块方法;共模漂移靠 **D5 机理锚点**。
- `floor_freeze`(DO_1_4 等):走 **⑥/⑦ freeze/floor** 逻辑,排除出白化类评分。

**按 ROI 排序的动手清单**:
1. **接 §1.1 残差/创新 + manifest**(改输入源 + 一个 `scoring_mode` 分支)——一次性消除 ❌②⑨ 与 ⚠️③⑧ 的根因。
2. **修两处非因果泄漏**:① `center=True`→因果窗;0.1 节 STL→改用 §1.1 残差(不再自带 STL)。
3. **④⑤ 去 `fillna(0)`**(改插值/掩蔽);④ 归一改 `std(resid)`。
4. **③ Page-Hinkley `δ/λ` 改 σ 倍数**。
5. **⑦/⑨ 先清 QR 负流量伪值**(§1.1 已打标),再算响应/变点。

**验收**:接入后,在 `iid` 通道上 ②⑨⑧ 的报警率应显著回落到接近名义水平;`autocorr_aware` 通道改由 ④⑤+D7 监控,不再由 ②⑨ 主导;floor 通道由 ⑥⑦ 处理。

## 4. 结论
- **稳健可直接用(原样)**:① Spike、④ Drift-PLS、⑤ FF-PCA、⑥/⑦ Freeze(其中 ① 建议修 `center=True`)。
- **必须先白化/对接 §1.1 才可信**:② Step-KS、⑨ PELT(❌);③ Page-Hinkley、⑧ Regime(⚠️,建议同改)。
- **分解**:可用但欠精,建议直接复用 §1.1 的分解结果而非 D1 自带固定 3 阶 / 有泄漏的 STL。
- 一句话:**D1 的多变量/局部稳健检测器(漂移/尖峰/冻结)设计良好、可用;真正受"无白化"之害的是依赖 i.i.d. 零分布的 Step-KS 与 PELT(及 Regime 的 KS 层)——这些应改在 §1.1 白化创新上运行,或对非白化通道改走多变量/n_eff 校正路径。**
